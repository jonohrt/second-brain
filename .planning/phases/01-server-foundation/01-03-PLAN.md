<task type="auto">
  <name>Task 1: Create Docker compose and SearXNG configuration</name>
  <files>docker/docker-compose.yml, docker/searxng/settings.yml</files>
  <action>
1. Create `docker/docker-compose.yml`:
   ```yaml
   services:
     searxng:
       image: searxng/searxng:latest
       container_name: searxng
       ports:
         - "8888:8080"
       volumes:
         - ./searxng/settings.yml:/etc/searxng/settings.yml:ro
       restart: unless-stopped
   ```
   Port 8888 externally to avoid conflicts with common services on 8080.

2. Create `docker/searxng/settings.yml`:
   ```yaml
   use_default_settings: true
   server:
     limiter: false
     secret_key: "<generate a random 32-char hex string>"
   search:
     formats:
       - html
       - json
   ```
   CRITICAL: The `json` format MUST be in the formats list or SearXNG returns 403 on `?format=json` requests. The `limiter: false` is required because this is a local-only instance and rate limiting would interfere with API usage.

3. Start the container: `cd docker && docker compose up -d`

4. Wait for container to be healthy (up to 30 seconds), then verify with:
   `curl -s "http://localhost:8888/search?q=test&format=json" | head -c 500`
   Confirm the response contains a `results` array.
  </action>
  <verify>
    <automated>curl -sf "http://localhost:8888/search?q=test&format=json" | python3 -c "import sys,json; d=json.load(sys.stdin); assert 'results' in d, 'No results key'; print(f'OK: {len(d[\"results\"])} results')"</automated>
  </verify>
  <done>SearXNG container running on port 8888, JSON API returns search results</done>
</task>
