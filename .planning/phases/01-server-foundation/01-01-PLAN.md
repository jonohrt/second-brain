<task type="auto" tdd="true">
  <name>Task 1: Extend config, create Fastify server with auth and /health</name>
  <files>src/types.ts, src/config.ts, src/server/index.ts, src/server/plugins/auth.ts, src/server/routes/health.ts, package.json</files>
  <behavior>
    - Config with server.port and server.apiToken parses correctly from YAML
    - Config without server section still loads (optional field)
    - GET /health returns 200 with { status: "ok", timestamp: string }
    - GET /health does NOT require auth
    - POST to any protected path without bearer token returns 401 with { error: "Unauthorized" }
    - POST to protected path with valid bearer token passes through
  </behavior>
  <action>
1. Install dependencies: `npm install fastify @fastify/bearer-auth`

2. Extend Config type in `src/types.ts` -- add optional `server` field:
   ```typescript
   server?: {
     port: number;
     apiToken: string;
   };
   ```

3. Update `src/config.ts` loadConfig() to parse server section from YAML. The YAML uses snake_case (`api_token`), map to camelCase (`apiToken`). Server section is optional -- do not throw if missing.

4. Create `src/server/routes/health.ts` -- export a Fastify plugin function `healthRoutes` that registers `GET /health` returning `{ status: "ok", timestamp: ISO string }`.

5. Create `src/server/plugins/auth.ts` -- export a Fastify plugin function `authPlugin` that takes `{ apiToken: string }` as options and registers `@fastify/bearer-auth` with:
   - `keys: new Set([apiToken])`
   - `errorResponse: () => ({ error: "Unauthorized" })` (generic per user decision)

6. Create `src/server/index.ts` with two exported functions:
   - `createApp(config: Config)`: Creates Fastify instance with `{ logger: true }`, registers /health OUTSIDE auth scope, then registers a scoped plugin that applies authPlugin + will receive protected routes. Returns the app instance. This function does NOT call listen() -- it just builds the app. This separation enables testing.
   - `startServer()`: Calls getConfig(), validates config.server exists and has apiToken (throw if not), calls createApp(config), then calls `app.listen({ port: config.server.port ?? 3000, host: '0.0.0.0' })`. Logs the Tailscale note: "Server listening on 0.0.0.0:{port} -- accessible via Tailscale".

7. Add `"server": "tsx src/server/index.ts"` to package.json scripts. The server/index.ts should call startServer() when run directly (check `import.meta.url` pattern or just add a top-level call guarded by a main check).

Note on INFRA-05 (Ollama memory): Add a comment in server/index.ts startup noting that OLLAMA_MAX_LOADED_MODELS=1 must be set as an env var for the Ollama process. No code changes needed -- Ollama handles this natively. The setting will be enforced via documentation in the SUMMARY.
  </action>
  <verify>
    <automated>npx tsc --noEmit</automated>
  </verify>
  <done>Config type extended, Fastify server builds without errors, /health route and auth plugin exist as separate modules, createApp is testable without listening</done>
</task>
