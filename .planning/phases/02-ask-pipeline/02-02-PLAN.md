  <action>
    Create src/services/ask-pipeline.ts:

    Export types:
    ```
    export interface AskConfig {
      similarityThreshold: number;  // default 0.65
      maxBrainResults: number;      // default 5
      maxWebResults: number;        // default 5
    }

    export type Source =
      | { type: 'vault'; path: string; title: string; similarity: number }
      | { type: 'web'; url: string; title: string };

    export interface AskResult {
      answer: string;
      sources: Source[];
      route: 'brain' | 'web' | 'both';
      model: string;
    }
    ```

    Export class AskPipeline:
    - Constructor takes: ollamaChat (OllamaChatService), searxng (SearxngService), embeddings (EmbeddingsService), supabase (SupabaseService), config (AskConfig with defaults).
    - async ask(question: string): Promise<AskResult>:
      1. Classify: call ollamaChat.classify(question) to get route.
      2. Retrieve brain context (if route is brain or both):
         - Embed question via embeddings.embed(question)
         - Search via supabase.searchWithScores(embedding, { threshold: config.similarityThreshold, limit: config.maxBrainResults })
         - If brain route and no results found, upgrade route to "web"
      3. Retrieve web context (if route is web or both):
         - Search via searxng.search(question, { limit: config.maxWebResults })
      4. Build generation prompt using buildGenerationPrompt helper (from research code example). System prompt includes assembled context. User message is the question.
      5. Generate answer via ollamaChat.chatWithFallback(messages)
      6. Assemble sources: map brain results to { type: 'vault', path: entry.vaultPath, title: entry.title, similarity } and web results to { type: 'web', url, title }.
      7. Return { answer: result.content, sources, route, model: result.model }.

    - Handle edge cases:
      - If embeddings.embed throws (Ollama down), catch and fall back to web-only route
      - If searxng.search throws, catch and continue with brain-only results (or empty context)
      - If no context from either source, generate with a system prompt saying "Answer based on your general knowledge. Note that no personal notes or web results were found."

    Create tests/services/ask-pipeline.test.ts:
    - Mock all four service dependencies with vi.fn()
    - Test brain route: mock classify->"brain", mock embed->vector, mock searchWithScores->results, mock chatWithFallback->answer. Verify sources have type "vault".
    - Test web route: mock classify->"web", mock searxng.search->results, mock chatWithFallback->answer. Verify sources have type "web". Verify embed NOT called.
    - Test both route: mock classify->"both", provide both brain and web results. Verify mixed sources.
    - Test brain-to-web fallback: mock classify->"brain", mock searchWithScores->[] (empty). Verify searxng.search IS called (route upgraded to web).
    - Test embeddings failure fallback: mock classify->"brain", mock embed to throw. Verify falls back to web search.
    - Test answer includes model name from chatWithFallback result.
