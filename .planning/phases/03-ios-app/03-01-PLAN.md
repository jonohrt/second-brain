  <action>
1. **ios/SecondBrain/Services/APIClient.swift**:
   - Struct `APIClient` with `baseURL: URL` and `apiToken: String` properties
   - Convenience init reading from `AppConfig` (default values)
   - Init accepting explicit baseURL and apiToken (for testing)
   - Private helper `func performRequest<T: Decodable>(method:path:body:) async throws -> T` that:
     - Builds URLRequest with baseURL + path
     - Sets Content-Type: application/json
     - Sets Authorization: Bearer {apiToken}
     - Sets timeoutInterval from AppConfig.requestTimeout
     - Encodes body with JSONEncoder
     - Calls URLSession.shared.data(for:)
     - Catches URLError and wraps as APIError.networkError or .serverUnreachable
     - Checks HTTP status code: if not 200, tries to decode APIErrorResponse for message, throws APIError.requestFailed(statusCode:message:)
     - Decodes response as T, catches DecodingError and wraps as APIError.decodingError
   - `func ask(text: String) async throws -> AskResponse` -- calls performRequest with POST /ask
   - `func capture(text: String, title: String? = nil, type: String? = nil, tags: [String]? = nil) async throws -> CaptureResponse` -- calls performRequest with POST /capture

2. **ios/SecondBrainTests/APIClientTests.swift**:

   - MockURLProtocol stores a static `requestHandler: ((URLRequest) throws -> (HTTPURLResponse, Data))?` closure
   - Create a URLSession with MockURLProtocol registered in its configuration
   - Inject this custom URLSession into APIClient (add an optional `urlSession` parameter to APIClient init, defaulting to .shared)
   - Tests per the behavior list above: verify correct decoding on success, verify correct error types on failure
